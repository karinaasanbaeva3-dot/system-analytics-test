# Задание 3: Архитектура PUSH-уведомлений

## Блок-схема отправки PUSH-уведомлений

```mermaid
graph TD
    A[Мобильное приложение открыто] --> B[Запрос разрешения на push]
    B --> C[Пользователь разрешает]
    C --> D[Получение push-токена от FCM/APNS]
    D --> E[Отправка токена на бэкенд]
    E --> F[Notification Service сохраняет токен]
    
    G[Cart Service: корзина брошена] --> H[Event Bus Kafka]
    I[Order Service: отмена заказа] --> H
    J[Promo Service: реклама] --> H
    H --> K[Notification Service получает событие]
    
    K --> L[Формирует текст уведомления]
    L --> M[Находит push-токен пользователя]
    M --> N{Какая платформа?}
    
    N -->|Android| O[Отправка в FCM]
    N -->|iOS| P[Отправка в APNS]
    
    O --> Q[Доставка на устройство]
    P --> Q
    
    Q --> R[Отображение уведомления]
    R --> S[Пользователь нажимает]
    S --> T[Открывается экран приложения]
## Схема обработки ошибок

```mermaid
graph TD
    A[Отправка push-уведомления] --> B{Успешно?}
    B -->|Да| C[Уведомление доставлено]
    B -->|Нет| D[Повтор через 5 секунд]
    D --> E{Попытка 2 успешна?}
    E -->|Да| C
    E -->|Нет| F[Повтор через 5 секунд]
    F --> G{Попытка 3 успешна?}
    G -->|Да| C
    G -->|Нет| H[Сохранить в Dead Letter Queue]
    H --> I[Отправить алерт в мониторинг]
```
Типы PUSH-уведомлений
Тип	Приоритет	Когда отправлять
Отмена заказа	Высокий	Сразу
Статус доставки	Высокий	Сразу
Подтверждение оплаты	Высокий	Сразу
Корзина брошена	Средний	Через 1 час бездействия
Рекламная рассылка	Низкий	По расписанию (утро/вечер)
Компоненты архитектуры
Компонент	Назначение
Notification Service	Главный сервис отправки пушей, формирует уведомления
Token Registry	Хранит push-токены пользователей (user_id → push_token)
Schedule Service	Планировщик для рекламных и утренних/вечерних пушей
Event Bus (Kafka)	Очередь событий от других микросервисов
FCM / APNS	Внешние платформы для доставки на устройства
Структура Token Registry
device_id	user_id	push_token	platform	timezone
Уникальный ID	ID пользователя	Токен для отправки	ios/android	Europe/Moscow

## Простой алгоритм:

1. **Нажмите** на текст выше и подержите — появится выделение
2. **Потяните** мышь вниз до самого конца
3. **Нажмите правую кнопку мыши** → **Копировать** (или Ctrl+C)
4. **Перейдите в GitHub** в поле редактирования
5. **Нажмите правую кнопку мыши** → **Вставить** (или Ctrl+V)
6. **Нажмите зеленую кнопку "Commit changes"**

---
