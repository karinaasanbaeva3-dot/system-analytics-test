## Блок-схема 2: Отправка push-уведомления по событию

┌─────────────────────────────────────────────────────────────────┐
│                    СЕРВИСЫ-ИСТОЧНИКИ СОБЫТИЙ                     │
│                                                                  │
│  Cart Service ──→ "Корзина брошена"                              │

│  Promo Service ─→ "Рекламная акция"                              │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                               ↓
┌─────────────────────────────────────────────────────────────────┐
│                    EVENT BUS (KAFKA)                            │
│                                                                  │
│  Очередь событий. Все сервисы публикуют события сюда.           │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                               ↓
┌─────────────────────────────────────────────────────────────────┐
│                    NOTIFICATION SERVICE                         │
│                                                                  │
│  1. Получает событие из Kafka                                    │
│  2. Определяет тип события (отмена заказа / реклама / и т.д.)    │
│  3. Формирует текст уведомления из шаблона                       │
│  4. Ищет push_token пользователя в Token Registry                │
│  5. Определяет платформу (Android → FCM, iOS → APNS)             │
│  6. Отправляет запрос на внешний сервис                          │
└──────────────────────────────┬──────────────────────────────────┘
                               │
                ┌──────────────┴──────────────┐
                ↓                             ↓
┌───────────────────────────┐   ┌───────────────────────────┐
│   FCM (Firebase)          │   │   APNS (Apple)            │
│   Для Android             │   │   Для iOS                 │
└───────────────┬───────────┘   └───────────────┬───────────┘
                │                               │
                └───────────────┬───────────────┘
                                ↓
┌─────────────────────────────────────────────────────────────────┐
│                    МОБИЛЬНОЕ ПРИЛОЖЕНИЕ                          │
│                                                                  │
│  1. Получает push-уведомление                                    │
│  2. Отображает в шторке уведомлений                              │
│  3. Пользователь нажимает на уведомление                         │
│  4. Открывается соответствующий экран                            │
└─────────────────────────────────────────────────────────────────┘

## Типы PUSH-уведомлений

| Тип | Приоритет | Когда отправлять |
|-----|-----------|------------------|
| Отмена заказа | Высокий | Сразу |
| Статус доставки | Высокий | Сразу |
| Подтверждение оплаты | Высокий | Сразу |
| Корзина брошена | Средний | Через 1 час бездействия |
| Рекламная рассылка | Низкий | По расписанию |

## Компоненты архитектуры

| Компонент | Назначение |
|-----------|------------|
| Notification Service | Главный сервис отправки пушей |
| Token Registry | Хранит push-токены пользователей |
| Schedule Service | Планировщик для пушей по расписанию |
| Event Bus (Kafka) | Очередь событий от микросервисов |
| FCM | Firebase для Android |
| APNS | Apple Push для iOS |

## Структура Token Registry

| device_id | user_id | push_token | platform | timezone |
|-----------|---------|------------|----------|----------|
| Уникальный ID | ID пользователя | Токен для отправки | ios/android | Europe/Moscow |
