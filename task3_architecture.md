# Задание 3: Архитектура PUSH-уведомлений

## Схема архитектуры

Мобильное приложение → регистрация токена → Backend (Notification Service) → FCM/APNS → мобильное приложение

## Компоненты:

1. **Notification Service** - главный сервис отправки пушей
2. **Token Registry** - хранит push-токены пользователей
3. **Schedule Service** - планировщик для утренних/вечерних пушей
4. **Event Bus (Kafka)** - очередь событий

## Типы пушей:

| Тип | Когда отправлять |
|-----|------------------|
| Отмена заказа | сразу |
| Корзина брошена | через 1 час |
| Реклама | по расписанию |

## Обработка ошибок:
- Если пуш не доставлен → повторить 3 раза
- Если не помогло → сохранить в очередь и отправить алерт

## Структура хранения токенов:

- device_id
- user_id
- push_token
- platform (ios/android)
- timezone
